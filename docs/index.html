ULSA Préstamos

Sistema de Préstamos ULSA

Plataforma web moderna para la gestión eficiente de préstamos de equipos y materiales universitarios

Objetivo del Proyecto

Desarrollar una solución integral que digitalice y optimice el proceso de préstamos en la Universidad La Salle, proporcionando una experiencia fluida tanto para estudiantes como para administradores, con autenticación segura y seguimiento en tiempo real.

Casos de Uso

Estudiante - Solicitar Préstamo

Actor: Estudiante ULSA

Descripción: Un estudiante desea solicitar el préstamo de un equipo disponible

Flujo:

  1. Estudiante accede al sistema con su cuenta institucional
  2. Navega al catálogo de equipos disponibles
  3. Selecciona el equipo deseado y especifica duración
  4. Sistema verifica disponibilidad y estado del estudiante
  5. Se genera la solicitud y notifica al administrador
  6. Administrador aprueba y registra la entrega

Administrador - Gestionar Inventario

Actor: Personal Administrativo

Descripción: El administrador gestiona el inventario y controla los préstamos

Flujo:

  1. Administrador accede al panel de control
  2. Visualiza estado en tiempo real de todos los equipos
  3. Puede agregar, modificar o dar de baja equipos
  4. Gestiona solicitudes pendientes de aprobación
  5. Controla devoluciones y registra el estado de los equipos
  6. Genera reportes de uso y disponibilidad

Sistema - Recordatorio Automático

Actor: Sistema Automatizado

Descripción: El sistema envía recordatorios automáticos sobre devoluciones pendientes

Flujo:

  1. Sistema verifica diariamente fechas de vencimiento
  2. Identifica préstamos próximos a vencer (2 días antes)
  3. Envía notificación por correo al estudiante
  4. Registra el recordatorio en el historial
  5. Si hay retraso, notifica también al administrador
  6. Actualiza el estado del préstamo a "vencido" si es necesario

Estudiante - Consultar Adeudos

Actor: Estudiante ULSA

Descripción: Un estudiante consulta su historial y adeudos pendientes

Flujo:

  1. Estudiante accede a su perfil personal
  2. Visualiza historial completo de préstamos
  3. Identifica préstamos activos y fechas de vencimiento
  4. Puede solicitar extensión de plazo si está disponible
  5. Recibe información sobre multas o restricciones
  6. Puede contactar al administrador para resolver dudas

Seguridad del Sistema

Autenticación Microsoft

El sistema implementa dos métodos robustos de autenticación con Microsoft:

1. OAuth 2.0 con Azure AD

// Configuración Microsoft Provider const provider = new firebase.auth.OAuthProvider('microsoft.com'); provider.setCustomParameters({ tenant: 'lasalle-tenant-id', prompt: 'select_account' }); // Autenticación firebase.auth().signInWithPopup(provider) .then((result) => { const user = result.user; validateInstitutionalEmail(user.email); });

2. Validación de Correo Institucional

function validateInstitutionalEmail(email) { const allowedDomains = [ '@lasalle.mx', '@ulsa.mx', '@estudiantes.lasalle.mx' ]; return allowedDomains.some(domain => email.toLowerCase().endsWith(domain) ); }

FingerprintJS Pro

Implementación avanzada de identificación de dispositivos:

Características de Seguridad:

  • Prevención de Fraude: Detecta intentos de acceso sospechosos
  • Sesiones Múltiples: Controla accesos simultáneos
  • Geolocalización: Verifica ubicación de acceso
  • Historial de Dispositivos: Mantiene registro de dispositivos autorizados
// Inicialización FingerprintJS import FingerprintJS from '@fingerprintjs/fingerprintjs-pro'; const fp = await FingerprintJS.load({ apiKey: 'YOUR_API_KEY', region: 'us' }); // Obtener fingerprint del dispositivo const result = await fp.get(); const visitorId = result.visitorId; // Validar dispositivo validateDeviceAccess(visitorId, userEmail);

Flujo de Validación:

  1. Usuario inicia sesión
  2. Sistema captura fingerprint del dispositivo
  3. Compara con dispositivos autorizados
  4. Si es nuevo, requiere verificación adicional
  5. Registra el dispositivo tras aprobación
